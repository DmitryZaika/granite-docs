Zapier Integration — Website

How to connect a website lead form to Granite Manager using Zapier.

Zap Overview

This Zap captures leads submitted through your website form and sends them to the Granite Manager webhook as a structured JSON payload.

The workflow consists of two steps:

  1. Webhooks by Zapier (Catch Hook) — trigger
  2. Webhooks by Zapier (Custom Request) — action

Note: Unlike Make, Zapier does not require a separate JSON transformer step. Your website form sends data directly to the Catch Hook, and you map fields straight into the Custom Request payload.

Required Payload Fields

When configuring the request body, use the exact field names below. Only name is required — all other fields are optional but recommended for a complete lead record.

Required

Field Type Description
name string Lead's full name

Optional

Field Type Description
phone string Contact phone number
email string Contact email address
address string Street address
city string City
postal_code string ZIP / postal code
details string Additional project notes or form answers
referral_source string Where the lead came from (e.g. website)
campaign_name string Campaign name
adset_name string Ad set name
ad_name string Ad name
start_date string When the client plans to start the project
project_size string Estimated project size
remodel_type string Type of remodel
backsplash string Backsplash preference

stove_type string Stove type

contact_time string Preferred contact time
improve_offer string Offer improvement response
your_message string Free-text message from the lead
file string Attached file reference

Tip: Map as many optional fields as your website form collects. Fields you leave empty are sent as null and ignored by the API.

1. Trigger: Webhooks by Zapier (Catch Hook)

Action: Add a Webhooks by Zapier → Catch Hook trigger as the first step in your Zap.

Configuration:

  1. In your Zap editor, click the trigger step and select Webhooks by Zapier as the app.
  2. Choose Catch Hook as the trigger event.
  3. Click Continue — Zapier will generate a unique webhook URL. Copy this URL.
  4. Configure your website form (HTML form, WordPress plugin, or custom handler) to send a POST request to this Zapier webhook URL when a visitor submits the form. Include field names that match your form inputs (e.g., name, phone, email, city).
  5. Submit a test form entry, then click Test trigger in Zapier to pull in the sample data.

Result: The Zap triggers instantly every time a visitor submits the lead form on your website.

2. Action: Webhooks by Zapier (Custom Request)

Follow these steps to securely send your lead data to the CRM.

Step 1: Add the Action

In your Zap editor, click + to add an action step. Search for Webhooks by Zapier and select Custom Request.

Part 1: Connection & Endpoint Setup

Method: Select POST.

URL: Input the following endpoint path exactly:

https://cawv6iwjgxpk5fj2fchs6vc5vq0bycwp.lambda-url.us-east-2.on.aws/v1/webhooks/new-lead-form/{company_id}

Company ID: Manually replace {company_id} with your actual organization ID. Ask your company administrator for your Company ID — you can find it in Admin → Company Settings → Company within Granite Manager.

Data Pass-Through? Select No.

Unflatten: Select No.

Part 2: Headers & Authentication

Click Show options to expand additional fields, then add a header:

Name Value
Authorization Bearer YOUR_API_TOKEN

API Token: Replace YOUR_API_TOKEN with the secret security token requested directly from the Granite Manager company.

Part 3: Body & Field Mapping

Payload Type: Select Json.

Under Data, add a row for each field from the Required Payload Fields section. Map Catch Hook values from the trigger step, or enter static text where noted:

API field Map from webhook
name (required) Full name field from form submission
phone Phone field
email Email field
city City field
postal_code ZIP / postal code field
details Project notes / additional form answers (combine multiple fields if needed)
your_message Message field
referral_source Type website as a static value
  • Dynamic fields: Click each value field and select the corresponding token from the Catch Hook trigger (e.g., contact details, project notes, message).
  • Static fields: For fields that need the same value on every lead (such as referral_source), type the plain text directly (e.g., website).

Wrap Request In Array: Leave No.

Part 4: Test & Publish

Click Continue, then Test step to send a sample lead to the webhook.

If the test returns a 201 response, click Publish to turn on the Zap.

Result: Sends the structured lead data directly into your CRM database webhook.
