# Make Integration — Website

How to connect a website lead form to Granite Manager using [Make](https://www.make.com) (formerly Integromat).

---

## Scenario Overview

This scenario captures leads submitted through your website form, formats them into a standardized JSON payload, and sends them to the Granite Manager webhook.

![Make.com scenario: Webhooks → JSON → HTTP POST](https://granite-database.s3.us-east-2.amazonaws.com/static-images/Screenshot+2026-06-27+195504.png)

The workflow consists of three modules connected left to right:

1. **Webhooks** (Custom webhook) — trigger
2. **JSON** (Create JSON) — transformer
3. **HTTP** (Make a Request) — action

Your website form sends a POST request to the Make webhook URL on every submission. Make then forwards the mapped data to Granite Manager.

---

## Required Payload Fields

When building your Make data structure, use the exact field names below. Only **`name`** is required — all other fields are optional but recommended for a complete lead record.

### Required

| Field | Type | Description |
|---|---|---|
| `name` | string | Lead's full name |

### Optional

| Field | Type | Description |
|---|---|---|
| `phone` | string | Contact phone number |
| `email` | string | Contact email address |
| `address` | string | Street address |
| `city` | string | City |
| `postal_code` | string | ZIP / postal code |
| `details` | string | Additional project notes or form answers |
| `referral_source` | string | Where the lead came from (e.g. `website`) |
| `compaign_name` | string | Campaign name |
| `adset_name` | string | Ad set name |
| `ad_name` | string | Ad name |
| `start_date` | string | When the client plans to start the project |
| `project_size` | string | Estimated project size |
| `remodel_type` | string | Type of remodel |
| `backsplash` | string | Backsplash preference |
| `sink` | string | Sink preference |
| `stove_type` | string | Stove type |
| `tear_out` | string | Tear-out details |
| `contact_time` | string | Preferred contact time |
| `improve_offer` | string | Offer improvement response |
| `your_message` | string | Free-text message from the lead |
| `file` | string | Attached file reference |

> **Tip:** Map as many optional fields as your website form collects. Fields you leave empty are sent as `null` and ignored by the API.

## 1. Trigger: Webhooks (Custom webhook)

**Action:** Add a **Webhooks** → **Custom webhook** module as the first step in your Make scenario.

**Configuration:**

1. Click the webhook module and select **Add** to create a new webhook.
2. Copy the generated webhook URL — you will configure your website form to POST to this address.
3. Set the webhook to accept incoming data (Make will auto-detect the payload structure after the first test submission).

**Website form setup:** Configure your site form (HTML form, WordPress plugin, or custom handler) to send a `POST` request to the Make webhook URL when a visitor submits the form. Include field names that match your form inputs (e.g., `name`, `phone`, `email`, `city`).

**Result:** The scenario triggers instantly every time a visitor submits the lead form on your website.

---

## 2. Transformer: JSON (Create JSON)

Follow these steps to format the webhook data into the structured payload required by Granite Manager.

### Step 1: Open the Module

Click on the purple **JSON (Create JSON)** module in your Make scenario.

### Step 2: Choose or Create a Data Structure

Click **Add** next to the "Data structure" field. Define a new schema using the field names from the [Required Payload Fields](#required-payload-fields) section above. At minimum, include `name`. Add any optional fields your website form collects (e.g., `phone`, `email`, `city`, `details`). Save the structure.

### Step 3: Map the Website Form Data

The module will display input fields generated by your data structure. Click into each field and map the corresponding dynamic tokens from the Webhooks trigger:

| API field | Map from webhook |
|---|---|
| `name` *(required)* | Full name field from form submission |
| `phone` | Phone field |
| `email` | Email field |
| `city` | City field |
| `postal_code` | ZIP / postal code field |
| `details` | Project notes / additional form answers |
| `your_message` | Message field |
| `referral_source` | Type `website` as a static value |

### Step 4: Save the Configuration

Click **OK** to save. The module will now automatically transform every incoming website submission into a clean, standardized JSON object ready for the HTTP request.

## 3. Action: HTTP (Make a Request)

Follow these steps to securely send your JSON lead data to the CRM.

### Step 1: Open the Module

Click on the blue **HTTP (Make a request)** module in your Make scenario.

### Part 1: Connection & Endpoint Setup

**Authentication type:** Select **No authentication** from the dropdown menu.

**URL:** Input the following endpoint path exactly:

`https://cawv6iwjgxpk5fj2fchs6vc5vq0bycwp.lambda-url.us-east-2.on.aws/v1/webhooks/new-lead-form/{company_id}`

> **Company ID:** Manually replace `{company_id}` with your actual organization ID. Ask your company administrator for your Company ID — you can find it in **Admin → Company Settings → Company** within Granite Manager.
>
> <img src="https://granite-database.s3.us-east-2.amazonaws.com/static-images/16be145e-f48c-44f8-abfd-13ef268b63f2.jpg" alt="Company ID location in Admin Settings" width="423" />

**Method:** Select **POST** from the dropdown menu. Ensure the adjacent **Map** toggle remains disabled.

**Headers:** Leave the **Map** toggle disabled. Click **Add a header** to configure Header 1:

| Name | Value |
|---|---|
| `Authorization` | `Bearer YOUR_API_TOKEN` |

> **API Token:** Replace `YOUR_API_TOKEN` with the secret security token requested directly from the Granite Manager company.

**Query parameters:** Leave this section completely empty. Do not click **Add a parameter**, and keep the **Map** toggle disabled.

**Body content type:** Select **application/json** from the dropdown menu.

**Body input method:** Select **Data structure** from the dropdown menu.

### Part 2: Body Structure & Field Mapping

**Body structure:** Click the dropdown and select your lead data structure. If it is not listed, click the **Add** button to create a new schema using the field names from the [Required Payload Fields](#required-payload-fields) section. The structure must include at least `name`.

**Body content:** Once the data structure is selected, the input fields will automatically generate below it. Map your incoming webhook fields to these inputs using the same field names as the API:

- **Dynamic fields:** Click inside each empty field and select the corresponding data tokens from the Webhooks or JSON module (e.g., `name`, `phone`, `email`, `details`).
- **Static fields:** If a field requires a fixed value for every incoming lead (such as `referral_source`), type the plain text directly into the input box (e.g., `website`).

### Part 3: Response & Advanced Settings

**Parse response:** Leave the **Map** toggle disabled and select the **Yes** radio button option.

**Pagination type:** Leave this field at its default placeholder state, **Select pagination type** (do not select any dropdown options).

**Advanced settings:** Keep this toggle switch turned off (greyed out).

**Save:** Click the purple **Save** button in the bottom right corner to complete the module configuration.

**Result:** Sends the structured lead data directly into your CRM database webhook.
