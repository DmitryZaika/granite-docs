# Make Integration — Facebook

How to connect Facebook Lead Ads to Granite Manager using [Make](https://www.make.com) (formerly Integromat).

---

## Scenario Overview

This scenario captures new Facebook leads, formats them into a standardized JSON payload, and sends them to the Granite Manager webhook.

![Make.com scenario: Facebook Lead Ads → JSON → HTTP POST](https://granite-database.s3.us-east-2.amazonaws.com/static-images/Screenshot+2026-06-27+191405.png)

The workflow consists of three modules connected left to right:

1. **Facebook Lead Ads** (New Lead) — trigger
2. **JSON** (Create JSON) — transformer
3. **HTTP** (Make a Request) — action

---

## Required Payload Fields

When building your Make data structure, use the exact field names below. Only **`name`** is required — all other fields are optional but recommended for a complete lead record.

### Required

| Field | Type | Description |
|---|---|---|
| `name` | string | Lead's full name |

### Optional

| Field | Type | Description |
|---|---|---|
| `phone` | string | Contact phone number |
| `email` | string | Contact email address |
| `address` | string | Street address |
| `city` | string | City |
| `postal_code` | string | ZIP / postal code |
| `details` | string | Additional project notes or form answers |
| `referral_source` | string | Where the lead came from (e.g. `facebook`) |
| `campaign_name` | string | Facebook campaign name |
| `adset_name` | string | Facebook ad set name |
| `ad_name` | string | Facebook ad name |
| `start_date` | string | When the client plans to start the project |
| `project_size` | string | Estimated project size |
| `remodel_type` | string | Type of remodel |
| `backsplash` | string | Backsplash preference |
| `sink` | string | Sink preference |
| `stove_type` | string | Stove type |
| `tear_out` | string | Tear-out details |
| `contact_time` | string | Preferred contact time |
| `improve_offer` | string | Offer improvement response |
| `your_message` | string | Free-text message from the lead |
| `file` | string | Attached file reference |

> **Tip:** Map as many optional fields as your Facebook Lead Form collects. Fields you leave empty are sent as `null` and ignored by the API.

## 1. Trigger: Facebook Lead Ads (New Lead)

**Action:** Connect your Facebook Business Account.

**Configuration:** Select the specific Facebook Page and the Lead Generation Form you want to track.

**Result:** The scenario triggers instantly every time a prospective client submits a new lead form.

---

## 2. Transformer: JSON (Create JSON)

Follow these steps to format the Facebook data into the structured payload required by Granite Manager.

### Step 1: Open the Module

Click on the purple **JSON (Create JSON)** module in your Make scenario.

### Step 2: Choose or Create a Data Structure

Click **Add** next to the "Data structure" field. Define a new schema using the field names from the [Required Payload Fields](#required-payload-fields) section above. At minimum, include `name`. Add any optional fields your Facebook form collects (e.g., `phone`, `email`, `city`, `details`). Save the structure.

### Step 3: Map the Facebook Lead Data

The module will display input fields generated by your data structure. Click into each field and map the corresponding dynamic tokens from the Facebook Lead Ads trigger:

| API field | Map from Facebook |
|---|---|
| `name` *(required)* | Full name |
| `phone` | Phone number |
| `email` | Email |
| `city` | City |
| `postal_code` | Zip code |
| `details` | Project notes / form answers (combine multiple fields with `;` if needed) |
| `compaign_name` | Campaign name |
| `adset_name` | Adset name |
| `ad_name` | Ad name |
| `start_date` | When are you planning to start the project? |
| `referral_source` | Type `facebook` as a static value |

### Step 4: Save the Configuration

Click **OK** to save. The module will now automatically transform every incoming Facebook lead into a clean, standardized JSON object ready for the HTTP request.

---

## 3. Action: HTTP (Make a Request)

Follow these steps to securely send your JSON lead data to the CRM.

### Step 1: Open the Module

Click on the blue **HTTP (Make a request)** module in your Make scenario.

### Part 1: Connection & Endpoint Setup

**Authentication type:** Select **No authentication** from the dropdown menu.

**URL:** Input the following endpoint path exactly:

`https://cawv6iwjgxpk5fj2fchs6vc5vq0bycwp.lambda-url.us-east-2.on.aws/v1/webhooks/new-lead-form/{company_id}`

> **Company ID:** Manually replace `{company_id}` with your actual organization ID. Ask your company administrator for your Company ID — you can find it in **Admin → Company Settings → Company** within Granite Manager.
>
> <img src="https://granite-database.s3.us-east-2.amazonaws.com/static-images/16be145e-f48c-44f8-abfd-13ef268b63f2.jpg" alt="Company ID location in Admin Settings" width="423" />

**Method:** Select **POST** from the dropdown menu. Ensure the adjacent **Map** toggle remains disabled.

**Headers:** Leave the **Map** toggle disabled. Click **Add a header** to configure Header 1:

| Name | Value |
|---|---|
| `Authorization` | `Bearer YOUR_API_TOKEN` |

> **Bearer token:** Ask the manager of the company you work with to go to **Admin → Company Settings → Company** in Granite Manager and click **Request Bearer**. They must then enter the email address where they would like to receive the bearer token.
>
> <img src="https://granite-database.s3.us-east-2.amazonaws.com/static-images/bearer+request.jpg" alt="Request Bearer button in Company settings" width="423" />

**Query parameters:** Leave this section completely empty. Do not click **Add a parameter**, and keep the **Map** toggle disabled.

**Body content type:** Select **application/json** from the dropdown menu.

**Body input method:** Select **Data structure** from the dropdown menu.

![HTTP module: authentication, URL, method, headers, and body settings](https://granite-database.s3.us-east-2.amazonaws.com/static-images/Screenshot+2026-06-27+193411.png)

### Part 2: Body Structure & Field Mapping

**Body structure:** Click the dropdown and select your lead data structure. If it is not listed, click the **Add** button to create a new schema using the field names from the [Required Payload Fields](#required-payload-fields) section. The structure must include at least `name`.

**Body content:** Once the data structure is selected, the input fields will automatically generate below it. Map your incoming Facebook fields to these inputs using the same field names as the API:

- **Dynamic fields:** Click inside each empty field and select the corresponding data tokens from your Facebook Lead Ads trigger (e.g., `name`, `phone`, `email`, `details`, `compaign_name`).
- **Static fields:** If a field requires a fixed value for every incoming lead (such as `referral_source`), type the plain text directly into the input box (e.g., `facebook`).

![HTTP module: body structure and field mapping](https://granite-database.s3.us-east-2.amazonaws.com/static-images/Screenshot+2026-06-27+192921.png)

### Part 3: Response & Advanced Settings

**Parse response:** Leave the **Map** toggle disabled and select the **Yes** radio button option.

**Pagination type:** Leave this field at its default placeholder state, **Select pagination type** (do not select any dropdown options).

**Advanced settings:** Keep this toggle switch turned off (greyed out).

**Save:** Click the purple **Save** button in the bottom right corner to complete the module configuration.

![HTTP module: parse response and pagination settings](https://granite-database.s3.us-east-2.amazonaws.com/static-images/Screenshot+2026-06-27+193446.png)

**Result:** Sends the structured lead data directly into your CRM database webhook.
